# Emergency Restore — Known Working Version 3

This package rolls the website back from the broken Version 4 test build to the last known working Version 3 foundation.

## Upload

Upload every file in this folder to the repository root and replace files with matching names.

Important: do not upload the ZIP itself and do not upload the enclosing folder.

After GitHub Pages finishes deploying:

1. Open the website in Safari.
2. Refresh twice.
3. Close Safari completely.
4. Reopen the website.
5. Remove and recreate the Home Screen shortcut only if Safari still shows Version 4.

The cache name is new, so the repaired build should replace the broken offline files.
